## 2024-05-22 - [Search Optimization Bottlenecks]
**Learning:** In Convex actions handling large datasets (up to 10k items), redundant lowercasing and array conversions (e.g., `Array.from(Set)`) inside loops create significant CPU bottlenecks. Additionally, fetching global metadata like "all supplier categories" on every request scales poorly. Sequential `await` in loops for database queries (RTT stacking) and O(N) array searches (like `.find()` or Map entry iteration) inside large loops are primary performance killers.
**Action:** Pre-calculate array forms of Sets, implement memoization for repetitive lookups like `isServiceCategory`, and refine database queries to target only the categories present in the current result set. Parallelize independent database queries using `Promise.all` and replace O(N) lookups with O(1) Map indexing or direct array access.
